真实案例 个人助理搭建

个人助理搭建指南

从零成本到专业级——5 个梯度，找到适合你的 AI 个人助理方案

核心判断

Claude Code Web = 云端沙箱，异步任务很好用；真正的全天候个人助理需要额外基础设施。

移动端现实

手机适合"下达任务、查看结果"，不适合"盯着 Agent 一步步操作"。把两者分开设计。

选择原则

从最简单的梯度开始，感受到瓶颈再升级——不要一开始就搭最复杂的方案。

梯度 1

零配置入门

零成本 即开即用 移动友好
架构：Claude.ai App（iOS/Android）+ Projects 功能

适合谁

刚开始探索 AI 个人助理、还不确定自己需要什么的人。

搭建复杂度

下载 App → 创建 Project → 写入个人背景信息，10 分钟完成。

实际工作流

在 Project 里写入你的身份、工作背景、常用风格偏好
↓ Claude 每次对话自动读取这些背景
手机上随时发送灵感、任务、问题
↓ 获得个性化回复，无需每次重新解释自己是谁
把常用的 prompt 模板存入 Project，随时调取

瓶颈所在

  • 无法主动触发：必须你先发消息，它才能响应
  • 无法联动外部工具：不能自动发邮件、改文件、搜索网络
  • 对话结束后无记忆延续（需要依赖 Project 的持久化信息）
Claude.ai App Projects 功能 Anthropic 账号（免费或 Pro）
梯度 2

轻量云端 Agent

低成本 几分钟设置 移动可操作
架构：Claude Code on the Web（浏览器/手机端）+ GitHub 仓库
这是你刚刚体验的方案。Claude 在云端维护一个沙箱环境，连接你的 GitHub 仓库，可以异步完成代码任务。

适合谁

有 GitHub 项目、需要异步改代码或构建内容的人。手机端描述任务，Agent 自主完成，你过会儿来看结果。

搭建复杂度

创建 GitHub 仓库 → 在 claude.ai/code 连接仓库 → 开始对话。无需本地安装任何东西。

实际工作流举例

手机打开 claude.ai，描述任务："帮我在这个文档站加一个按标签筛选的功能"
↓ Agent 读取整个仓库，规划实现方案
Agent 自主写代码、测试、提交到仓库（你无需盯着）
↓ 任务完成后通知你
手机上查看结果，满意则合并；不满意可以继续对话修改

瓶颈所在

  • 沙箱权限有限：无法访问外部私有 API、无法持续运行后台进程
  • 不适合需要长时间运行的任务（超时会中断）
  • 手机端查看复杂 diff 体验较差，审查代码建议用桌面端
  • 非代码类任务（如自动发邮件、管理日历）能力有限
claude.ai/code GitHub 仓库 Claude Max 订阅（推荐）
梯度 3

自动化工作流

低成本 半天搭建 手机触达
架构：n8n（自托管）或 Zapier + Claude API → 飞书 / 微信 / 邮件推送

适合谁

想要"定时触发 + 跨应用联动"、不需要写代码的人。适合运营、产品、内容创作者。

搭建复杂度

注册 n8n cloud 或自部署 → 拖拽连接节点 → 配置 Anthropic API Key。核心流程半天内可完成。

实际工作流举例：每日简报助理

每天 8:00，n8n 定时触发
↓ 抓取你订阅的 RSS / 行业网站更新
Claude API 对内容进行筛选、摘要、按重要性排序
↓ 生成一份个性化早报
推送到你的飞书或微信（通过飞书机器人 Webhook）

另一个例子：邮件智能分类

新邮件到达 Gmail
↓ n8n 触发，将邮件内容传给 Claude
Claude 判断优先级、生成简短摘要
↓ 结果推送到手机飞书通知
重要邮件立刻提醒，普通邮件日报汇总

瓶颈所在

  • n8n 自托管需要一台服务器（但可以用最便宜的 VPS，约 30 元/月）
  • 每次调用 Claude API 有费用（通常很低，几分钱到几毛钱/次）
  • 复杂逻辑需要学习 n8n 节点配置，有一定学习曲线
n8n（推荐自托管） Anthropic API 飞书 / 企业微信 Webhook Gmail / RSS 等数据源
梯度 4

Mac Mini 常驻方案

中等成本 一天搭建 手机全控 推荐
架构：Mac Mini 24 小时运行 Claude Code CLI → Telegram / 飞书 Bot 作为手机交互入口
这是你提到的方向，也是目前个人用户搭建全天候 AI 助理最务实的路径。Mac Mini 功耗极低（约 6-10W 待机），7×24 静音运行，配合 Tailscale 在任何网络环境下都能从手机访问。

适合谁

对隐私有要求、想要最大灵活性、愿意花一天时间初始搭建的人。数据不出本地，模型调用可选 Anthropic API 或本地模型。

搭建复杂度

需要：购买 Mac Mini（M4 款约 4000 元起）→ 安装 Claude Code + 配置 API Key → 搭建 Telegram Bot → 配置 Tailscale 内网穿透。有一定技术门槛，但每步都有文档。

实际工作流举例

手机飞书/Telegram 发消息："把昨天会议录音整理成行动项，发给对应的人"
↓ Mac Mini 上的 Agent 收到指令
Agent 读取本地录音文件、转文字、提取行动项、起草邮件
↓ 任务完成，结果回传到手机
你在手机上确认，Agent 发送邮件

随时随地触达

需要注意

  • 初始硬件成本（Mac Mini M4 约 4000 元），但电费极低
  • 需要配置 Tailscale 或类似工具才能在外网访问到家里的 Mac Mini
  • Claude Code CLI 目前仍需要人类监督复杂任务，完全无人值守有风险
Mac Mini M4 Claude Code CLI Anthropic API Tailscale（内网穿透） Telegram Bot API 或飞书机器人
梯度 5

云主机专业级

按量付费 需要技术背景 任意入口
架构：VPS 云主机 + Claude Code / 多 Agent 编排 → 任意移动端入口

适合谁

开发者，需要 7×24 运行、多任务并发、或需要公网 IP 接收 Webhook 的场景。

搭建复杂度

购买 VPS（阿里云/腾讯云轻量版，2C4G 约 50-100 元/月）→ 部署 Agent 栈 → 配置接入层。需要 Linux 基础和 API 集成经验。

典型场景

公网 Webhook 接收 GitHub/Jira 事件（Mac Mini 方案做不到）
↓
多个 Agent 并发处理不同任务，互不干扰
↓
自定义 Web UI 或 App，不依赖第三方聊天工具

与梯度 4 的本质区别

  • 有公网 IP，可以直接接收外部推送（Webhook）
  • 资源可弹性扩展，不受单台机器限制
  • 数据在云端，隐私保护不如本地方案
  • 月度持续费用，需要评估 ROI
阿里云 / 腾讯云 轻量应用服务器 Claude Code 或 LangGraph Anthropic API Docker（可选） 任意 Bot 框架

加餐：iOS 捷径 + Claude API

不需要任何服务器，纯手机端最轻量的"触发 Agent"方案

怎么做

用 iOS 捷径（Shortcuts）构建工作流：语音输入 → 调用 Claude API → 结果读出来或存到备忘录。可以放到锁屏小组件，一键触发。

适合场景

快速灵感记录 + AI 整理、语音转待办、路上听到的内容快速提炼。真正做到"零摩擦"的移动端 AI 入口。

iOS Shortcuts Anthropic API Key 备忘录 / Notion / Obsidian
